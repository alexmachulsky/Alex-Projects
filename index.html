About

DevOps Engineer from Israel with hands-on experience designing and operating production infrastructure on AWS. I build end-to-end deployment pipelines — from provisioning EKS clusters with Terraform to shipping containerized workloads through GitOps workflows powered by ArgoCD.

My day-to-day work spans Docker, Kubernetes, Helm, and CI/CD automation with GitHub Actions and Jenkins. I manage secrets with AWS Secrets Manager and External Secrets Operator, enforce security scanning with Trivy, and keep systems observable through Prometheus, Grafana, and the ELK stack (Elasticsearch, Fluent Bit, Kibana).

I thrive on turning complex manual processes into repeatable, automated solutions — whether that means standing up a full-stack application on EKS, building intelligent chatbots with local LLMs, or creating fitness analytics platforms. Every project I ship is containerized, monitored, and deployed through a proper CI/CD pipeline.

Blackjack Game on AWS

A comprehensive DevOps project that deploys an interactive Python Flask blackjack game to AWS EKS using enterprise-grade tooling. Infrastructure is provisioned with Terraform, containers are orchestrated with Kubernetes and Helm, and deployments follow a GitOps workflow through ArgoCD. Secrets are managed via AWS Secrets Manager with External Secrets Operator, and the full observability stack includes Prometheus, Grafana, Elasticsearch, Fluent Bit, and Kibana. MongoDB serves as the data layer.

AI / DevOps

AI ChatBot

An AI-powered chatbot running 100 % locally with Ollama and Llama 3.1 8B — no external API keys required. Features streaming responses, a document-grounded RAG mode, optional Google web lookup, and a switchable model picker. The Flask backend is served by Gunicorn, orchestrated with Docker Compose alongside the Ollama server, and deployable to Kubernetes with liveness/readiness probes and rolling updates. A GitHub Actions pipeline lints, builds, scans with Trivy, and publishes container images.

Smart Gym Progress Tracker

A production-grade fitness analytics platform built in phases with a DevOps-first approach. React (Vite) frontend paired with a FastAPI backend and PostgreSQL database. Features JWT authentication, multi-set workout logging, body-weight tracking, and progression intelligence (volume, estimated 1RM, plateau detection). Containerized with Docker Compose, tested with pytest, and published to GHCR via GitHub Actions. Phase 3 adds Terraform-provisioned AWS EKS and Kubernetes manifests for full cloud deployment.
